3. Getting Started

Prerequisites

  • Node.js v18 or higher
  • npm or yarn
  • API credentials for each service

Installation Steps

Step 1: Install Dependencies

# Frontend
npm install

# Backend
cd backend && npm install

Step 2: Configure API Credentials

Edit backend/config.js with your API keys and tokens.

Step 3: Start the Servers

# Backend (runs on port 3001)
cd backend && node server.js

# Frontend (runs on port 5173)
npm run dev

Step 4: Authorize Services

On first load, an OAuth modal will appear. Click "Authorize" for Google Services and complete the sign-in flow.

Quick Start URLs:
Frontend: http://localhost:5173
Backend: http://localhost:3001

5. API Integrations

Google Services (OAuth2)

All Google services share the same OAuth2 flow:

  1. User clicks "Authorize" in OAuth modal
  2. Popup opens to Google sign-in
  3. User grants permissions
  4. Tokens stored in backend memory
  5. Automatic token refresh when expired
Required OAuth Scopes:
  • webmasters.readonly - Search Console
  • analytics.readonly - Analytics
  • business.manage - My Business
  • adwords - Google Ads

7. Troubleshooting

Common Issues

"Authentication required" Error

Cause: OAuth tokens not set or expired
Solution: Refresh page and click "Authorize" in OAuth modal

Google Ads "UNIMPLEMENTED" Error

Cause: Developer token has "Test Access" level
Solution: Apply for "Basic Access" at Google Ads API Center

Google My Business Rate Limits

Cause: Too many API requests
Solution: Backend implements 30-minute caching and exponential backoff. Wait and retry.

"N/A" Displayed for Metrics

Cause: API returned no data or error
Solution: Check backend console logs, verify credentials, ensure APIs are enabled

Useful Commands

# Check backend health
curl http://localhost:3001/health

# Check OAuth status
curl http://localhost:3001/api/google/oauth/status

# Restart servers
lsof -ti:3001 | xargs kill -9  # Kill backend
lsof -ti:5173 | xargs kill -9  # Kill frontend

# View backend logs
tail -f /tmp/backend.log
